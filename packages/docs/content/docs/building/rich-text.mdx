---
title: Rich text
description: Flow styled, tappable text runs and inline widgets as one wrapping block with RichText
---

`RichText` lays out a mix of styled text runs and inline widgets as a single
wrapping paragraph — the Instagram-caption pattern: a tappable avatar and
`@username`, then the caption, all flowing together and wrapping with
`maxLines` / `overflow`. It maps to Flutter's `Text.rich`.

```tsx
import { RichText, TextSpan, View, Image } from "solid-fuse";

<RichText maxLines={4} overflow="ellipsis">
  {/* Any non-TextSpan child flows inline as a WidgetSpan */}
  <View width={18} height={18} borderRadius={9} clipBehavior="antiAlias">
    <Image src={user.avatarUrl} width={18} height={18} fit="cover" />
  </View>
  <TextSpan fontWeight="bold" color="#fff" onTap={() => openProfile(user.id)}>
    {" @" + user.handle + " "}
  </TextSpan>
  <TextSpan>{caption}</TextSpan>
</RichText>
```

## How children map to spans

`RichText` walks its children and turns each into an inline span:

| Child | Becomes |
|---|---|
| A `<TextSpan>` | a styled (optionally tappable) text run |
| A bare string | a text run in the root `RichText` style |
| Any other widget (`<View>`, `<Image>`, …) | a `WidgetSpan` that flows inline with the text |

The props on `RichText` itself set the **root style** — every run inherits it
unless a `TextSpan` overrides a property. `RichText` also owns the block-level
layout props: `maxLines`, `overflow`, `textAlign`, `softWrap`, and `textScaler`.

## Inline widgets flow with the text

Because any non-`TextSpan` child becomes a `WidgetSpan`, an avatar, badge, or
icon wraps inline alongside the words — it isn't a separate row. This is what
lets the avatar + handle + caption read as one continuous block that ellipsizes
as a whole.

Control how an inline widget sits against the text baseline with its
`alignment` prop (a `PlaceholderAlignment`):

```tsx
<RichText>
  <View width={16} height={16} alignment="middle">
    <Image src="badge.png" width={16} height={16} />
  </View>
  <TextSpan> verified</TextSpan>
</RichText>
```

`alignment` is one of `baseline`, `aboveBaseline`, `belowBaseline`, `top`,
`bottom`, or `middle` (the default).

## Tappable runs

Give a `TextSpan` an `onTap` to make just that run tappable — it's wired to a
Flutter `TapGestureRecognizer`, so the rest of the paragraph stays inert:

```tsx
<TextSpan fontWeight="bold" onTap={() => openProfile()}> @lysbth </TextSpan>
```

## Nesting

A `TextSpan`'s children may be a string, nested `<TextSpan>`s, or a mix. Nested
spans inherit their parent's style and override selectively:

```tsx
<RichText color="#fff">
  <TextSpan>
    Liked by <TextSpan fontWeight="bold" onTap={likers}>lysbth</TextSpan> and{" "}
    <TextSpan fontWeight="bold">12 others</TextSpan>
  </TextSpan>
</RichText>
```

## Styling props

Both `RichText` (as the root style) and `TextSpan` accept the same `TextStyle`
subset: `fontSize`, `fontWeight`, `fontFamily`, `fontStyle`, `color`,
`lineHeight`, `letterSpacing`, `wordSpacing`, `textDecoration`,
`textDecorationColor`, `textDecorationStyle`, `backgroundColor`, and `shadows`.

<Callout>
`TextSpan` is only valid inside a `RichText`. It's a marker the `RichText`
interprets — it doesn't render on its own.
</Callout>
