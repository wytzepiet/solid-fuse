Every built-in widget ships as an uppercase wrapper component imported from `solid-fuse`. They take typed props and render to native Flutter widgets — there's no DOM, no webview. `<View>` becomes a real Flutter `Container`, `<Text>` a real `Text`, and so on.

This section documents that catalogue, one widget (or closely related family) per page.

## Conventions

A few rules hold across every widget page, so they're stated once here rather than repeated:

- **Flutter names, not DOM names.** Wrappers are named after the Flutter widget they map to, and so are their props and methods (`requestFocus`, not `focus`). Each page links to the matching [Flutter API doc](https://api.flutter.dev/) so you can drop down to the underlying widget's full reference.
- **Every prop is reactive.** Drive any prop from a signal and the widget updates in place — Fuse diffs the props and Flutter rebuilds only what changed. Pages don't repeat this; assume it everywhere.
- **`flex` is universal.** Every element that accepts children takes a [`flex`](/docs/widgets/layout/flex) config for layout, instead of separate `Row`/`Column` components.
- **Shared input shapes** — colours, spacing, borders, decorations — are documented once under [Input types](/docs/widgets/reference/input-types) and referenced from each widget that accepts them.
